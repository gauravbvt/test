Viewing task assignments and sharing commitments

Recall that tasks are defined in terms of agents, roles, places, and organizations, and that these entities can be actual or types. A task specification may reference entity types or omit to identify by name an actual agent, role, jurisdiction, etc. so as to encompass more than one possible agent as executor of the task. A task thus defined could be assigned to more than one agent.

A planner can look up the list of known agents that are implicitly or explicitly assigned to the task (the task assignments).

Agents are implicitly bound by sharing commitments when they are assigned to tasks that are sources of shared information. All agents assigned to the "source" task have sharing commitments with all agents assigned to the "recipient" task, except:

  • if a prohibiting policy is attached to the sharing flow,

  • if a restriction applies - must be in same (top) organization or location - and a "source" agent and "recipient" agent do not meet this restriction.

A planner can look up the list of sharing commitments inferred by an information flow.
