Activities

An ISP Method Activity is carried out by a Planner to achieve an outcome that (ideally) brings a plan closer to completion. An Activity is composed of a number of steps, many or all optional, often carried out in no specific order. Most Activities can be undone (for example, a specified task can be later removed). When they exist, the prerequisites for an Activity are usually few and commonsensical; for example, a task in a segment can not be specified unless the segment is already defined.

The outcomes of an Activity can be completely or partially undone at any time. This is accomplished by removing a task specification, deleting a Plan segment, removing a member from an organization, reverting to a prior specification or definition etc.). Such “reverse Activities” are not listed but are usually implied.

Prior to applying any activtiy, it is assumed that:

  • a plan has been created and given a unique identifier (the identifier is not the name of the plan; it is universally unique and can not be changed)

  • users have been authorized as planners

  • a default segment is created for a default phase of a default incident event

Activities fall under scoping a plan, specifying information sharing procedures, profiling entities, or improving a plan.

Scoping a plan

The Scoping Activities are:

  • Localize the Plan

  • Name an anticipated event

  • Name a planned phase

  • Name a participating organization

  • Identify a participant

  • Add a segment

  • Set an organizational goal

Localize the Plan

Outcomes

Place types can now be defined relative to the plan’s reference location.

Prerequisites

The Plan was created.

Steps

  • Optionally name the reference location of the plan (an actual or type of place)

Name an anticipated event

Outcomes

An additional event must be planned for.

Prerequisites

The Plan was created.

Steps

  • Name the event

  • Indicate whether the event is an incident (there must be at least one incident per plan)

Name a planned phase

Outcomes

An additional phase must be planned for each event in scope.

Prerequisites

The Plan was created.

Steps

  • Name the phase

  • Identify whether it is pre-, co- or post-event

Name a participating organization

Outcomes

An organization is identified as an expected participant.

Prerequisites

The Plan was created.

Steps

  • Name the organization

  • Indicate that its participation is expected

  • Profile the organization

Identify a participant

Outcomes

An agent is identified as an expected participant.

Prerequisites

The organizations the agent is a participating member of are named.

Steps

  • Name the agent in the job roster of a participating organization

  • Give the job’s title, and name the role and, optionally, jurisdiction

  • Profile the role, agent and jurisdiction, if named

Add a segment

Outcomes

The plan can now cover another event phase or the tasks and flows that address an organizational goal for an event phase can be moved to the new segment.

Prerequisites

The Plan was created.

Steps

  • Create the segment, then

  • Name the segment

  • Name the event and phase it covers

Add an organizational goal

Outcomes

Mitigation of a risk to or capture of a gain by an organization is added to the scope.

Prerequisites

The segment to address the goal is created

Steps

  • Select risk mitigation or gain capture then, in any order,

  • Categorize the risk or gain

  • Set the level of risk or magnitude of the gain

  • Name the organization at risk or who would benefit

  • Describe the goal

Specifying procedures

The Specifying Activities are:

  • Specify a task

  • Specify an information flow

    • Specify an information need

    • Specify an information capability

    • Specify an information sharing

Specify a task

Outcomes

A task is added to a segment of the Plan and agents may be assigned to it.

Prerequisites

The containing segment is created.

Steps

  • Name the task

  • Categorize it (operations, operations management, operational coordination, direction, audit, analysis, preparing/planning or policy-setting)

    A task can be categorized as:

    • operations (primary activities, the work that directly accomplishes the mission of the organization)

    • operations management (directly regulating and measuring operations)

    • inter-operations coordination (avoiding and resolving operational conflicts but without involving higher ups)

    • direction (what the higher ups do: operational cohesion and optimization, resource allocation, setting performance targets)

    • audit (verification of operational performance measures by higher ups)

    • analysis (making sense of the external environment)

    • planning/preparing (forward looking activities: market research, forward planning, strategy, R&D)

    • policy-setting (the ultimate authority: mission definition and evolution, ethos, balancing the demands of now vs future).

  • Name the location of its execution

  • Name the actual or type of agent assigned to it

  • Name the role of the agents assigned

  • Name the jurisdiction for the role

  • Name the actual or type of organization the agents represent

  • Indicate whether assignees execute the task as a team or separately

  • Named the event the task causes, if any

  • Describe how long the task usually takes to complete, and whether it recurs and after how long

  • Indicate whether the task is immediate

  • Indicate whether successful execution of the task can terminate the phase of the event of the segment it is in

  • Identify the goals of this segment the task achieves, if any

  • Enter any instructions for the assignees

  • Attach/link and name policy an reference documents

Specify an information need, capability or flow

Outcomes

An information need, capability or sharing is added to a task.

Prerequisites

The task is added to the segment.

Steps

(All steps are optional and they can be taken in any order)

  • Name the information

  • Categorize its intent (alarm, announcement, command, feedback or report)

    The intent of a (needed, potential or committed-to) flow can be one of:

    • report (summary of facts and conjectures: "Here's what we know...")

    • command (an injunction: "Do this!" "Stop that!")

    • announcement (declaration, communiqué, advertisement: “I-95 is congested”, “Spinach is safe”)

    • feedback (thumbs up or down, reward, punishment: "Objectives were met", "Report is incomplete")

    • alarm (signal of an unexpected change requiring outside assistance: "Distribution point is overwhelmed!", “Levee is breached”)

  • State whether communication is to be initiated on task failure only (applies to "requested receives" and "sent notifications")

  • Name the information's elements and, if applicable,

    • identify their secrecy classifications

    • give a description

    • give handling instructions

    • if sent, specify relationship to received element(s), if any (identical, renamed, aggregated)

  • Indicate whether the information would be shared as a notification or as an answer to a request

  • Whether the information must go to/come from an agent in (restriction on commitments)

    • the same overall organization

    • the same organization

    • the same location

  • List the alternate transmission channels to be used

  • Indicate the impact on the task of sending or receiving the information

  • Give time constraints on sending or receiving the information

  • Enter instructions

  • Attach/link and name a policy (mandating, prohibiting, constraining) document, reference document or illustrative media file.

Specify an information need (extends Specify an information need, capability or flow)

Outcomes

An information need is added to a task.

Prerequisites

(as in Specify an information need, capability or flow)

Steps

(as in Specify an information need, capability or flow)

  • the impact of fulfilling the need can be to trigger the task or make a critical contribution to its success

  • the time constraint is the maximum delay between receiving the information and it becoming available (if notification) or it being requested (if answer to request)

Specify an information capability (extends Specify an information need, capability or flow)

Outcomes

An information capability is added to a task.

Prerequisites

(as in Specify an information need, capability or flow)

Steps

(as in Specify an information need, capability or flow) except

  • the impact of sharing the information produced could be to terminate the task

  • the time constraint is the maximum delay between producing the information and sharing it (if notification offered) or it being requested and sharing it (if answer to request offered)

Specify an information sharing (extends Specify an information need, capability or flow)

Outcomes

Information sharing commitments are specified between a producing task and a consuming task.

Prerequisites

(as in Specify an information need, capability or flow)

Steps

(as in Specify an information need, capability or flow) except

  • the source or target of the flow is identified (depending on whether the flow is looked at form the perspective of the target or source of the flow)

  • the impacts on both source and target tasks are given

Profiling entities

The Profiling Activities are:

  • Profile an event type

  • Profile a phase

  • Profile a place

  • Profile an organization

  • Profile a role

  • Profile an agent

  • Profile a transmission medium

Profile an event type (extends Profile an entity)

Outcomes

An entity referenced in the Plan is profiled, which depending on the kind of entity, may alter task assignments and thus information sharing commitments.

Prerequisites

The entity is referenced by name in the Plan either as an actual or type of entity (roles, phases and transmission media can only be types).

Steps

(All steps are optional and they can be taken in any order)

  • Name or change the name (a name must be unique for the kind of entity)

  • Give a description (free text)

  • Tag the entity with the names of entity types of the same kind (e.g. The actual place “Headquarters” is tagged with place type “Building”)

  • Attach/link reference and policy documents

Profile a phase (extends Profile an entity)

Outcomes

An event referenced in the Plan is profiled.

Prerequisites

(as in Profile an entity)

Steps

(Added to steps in Profile an entity)

  • Name a place (actual or type) where the event would occur

Profile a phase (extends Profile an entity)

Outcomes

Changing in mid-plan the meaning of a phase (changing it from pre-event to post-event, for example) can profoundly alter the meaning of the entire plan.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

  • Set the phase as either pre-event, co-event or post-event

Profile a place (extends Profile an entity)

Outcomes

Assignments may change for tasks that specify a jurisdiction, and thus sharing commitments as well.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

  • Name another actual place this place is within

  • If an actual place:

    • Give the street address, postal code, city, county, state and country

  • If a type of place:

    • Declare it to be inside either the plan’s locale, the location of an event or a named place (actual or type)

    • Declare it to contain either the plan’s locale, the location of an event or a named place (actual or type)

Profile an organization (extends Profile an entity)

Outcomes

Assignments may change for tasks that specify an organization, and thus sharing commitments as well.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

  • Describe the mission

  • Name an actual parent organization

  • If an actual organization:

    • Name the location of the organization (an actual place)

    • Give contact info for the organization as a whole (as endpoints on transmission media)

    • Declare that there should be no sharing commitment not covered by a sharing agreement

    • Declare that a role associated with the organization should be assigned to at least one agent

    • Define memberships in the organization (aka jobs):

      • Naming an agent (mandatory)

      • Giving a job title (mandatory)

      • Naming a role (mandatory)

      • Naming a jurisdiction (optional)

      • Naming a supervisor (optional)

    • For each sharing commitment by the organization, indicate if it is covered by a sharing agreement

Profile a role (extends Profile an entity)

Outcomes

Assignments may change for tasks that specify a role, and thus sharing commitments as well.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

Note that there is no such thing as an “actual role”, roles are all types

  • (No additional step)

Profile an agent (extends Profile an entity)

Outcomes

Assignments may change for tasks that specify an agent type, and thus sharing commitments as well.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

  • Declare whether or not the agent is an archetype

  • Declare whether the agent is a system or a person

  • Provide contact information as endpoints and transmission media

  • List security clearances

Profile a transmission medium (extends Profile an entity)

Outcomes

The applicability of a transmission medium may depend on its nature (unicast vs multicast vs broadcast) and its security classification. Changing the acceptable pattern for endpoints may render some channels invalid.

Prerequisites

(as in Profile an entity)

Steps

(Optionally added to steps in Profile an entity)

  • Declare the mode of transmission to be either unicast, multicast or broadcast

  • Specify acceptable endpoints as a string pattern

  • List other transmission media this one can delegate to

  • List the security level of the medium

Improving a Plan

The Improving Activities are:

  • Move tasks across segments

  • Generalize a task specification

  • Change an entity’s classification

Move tasks across segments

Outcomes

One segment is reduced in size and becomes more manageable. Goals may be added to the destination segment from moved tasks that achieve them.

Prerequisites

At least two segments are defined.

Steps

  • Remove one or more tasks from the segment of origin

  • Add them to the destination segment

  • Add, as needed, to the destination segment goals that are achieved by a moved task

Generalize a task specification

Outcomes

The number of assignments for the task grows as it matches more participating agents, and, consequently, so does the number of sharing commitments.

Prerequisites

The task is specified.

Steps

(Carry out one or more of these steps as applicable and in any order)

  • Remove the named agent from the task specification

  • Replace the actual agent by a type of agent

  • Replace the type of agent by one of the types it is tagged with

  • Remove the role

  • Replace the role by one of the roles it is tagged with

  • Remove the actual jurisdiction (a place)

  • Replace the type of jurisdiction by one of the types it is tagged with

  • Remove the named organization from the task specification

  • Replace the actual organization by a type of organization

  • Replace the type of organization by one of the types it is tagged with

Change an entity’s classification

Outcomes

The meaning of the entity changes. Now, different agents match the specification of tasks that reference the entity. Agents tagged with the entity get re-assigned to different tasks. As a consequence, sharing commitments also change.

Prerequisites

The entity is profiled and so are the entity types it is tagged with.

Steps

  • Add or remove tags to the entity’s profile

  • Or change the tags of entity types that define this entity.
