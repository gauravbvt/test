Actions

Planners can create and remove segments; add, cut, copy and paste tasks, needs, capabilities and flows, connect tasks with information flows, disconnect them, add an issue, remove a segment, move tasks across segments, add an intermediate to a flow etc. Relevant actions are all available from “action” menus within the context of a plan, segment, task, flow or entity.

Some actions have simple effects (adding a task, for example) but others can be more complex:

  • “disintermediate” removes a task that acts as a middleman in a flow

  • “intermediate” adds a middleman task to a flow,

  • “satisfy needs” automatically connects information needs of a task to matching but untapped information capabilities from other tasks,

  • "move tasks" moves one or more tasks from one segment to another, possibly also adding goals achieved by the moved tasks to the target segment.

Most actions can be undone and redone up and until some planner takes an action that would conflict with the undoing or redoing (for example, removing a task prevents undoing a change to one of its attributes).

These actions are available from the top actions menu:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about the current segment

  • Pasting a copied task into the current segment

  • Pasting a copied attachment to the current segment

  • Adding a new issue to the current segment

  • Adding a new segment to the Plan

  • Removing the current segment

  • Moving tasks en masse from this segment to another

  • Logging out

These actions are available from the Plan's definition panel:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about the Plan

  • Pasting a copied attachment to the Plan

  • Adding a new issue to the Plan

These actions are available from the segment's definition panel:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about the segment

  • Pasting a copied attachment to the segment

  • Adding a new issue to the segment

These actions are available from the edit form of a task:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about the task

  • Copying the task

  • Cutting the task (copy and remove), Note that the last task in a segment can not be removed.

  • Pasting a copied attachment to the task

  • Pasting (the specifications of a copied task)

  • Adding a new issue to the task

  • Adding a duplicate of this task to the segment

  • Satisfying all (satisfiable) information needs of the task

  • Disintermidiating (bypassing the task with flows where the task acts as middleman)

These actions are available from the edit form of an information need, capability or sharing flow:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about it

  • Copying it

  • Pasting a copied attachment to it

  • Duplicating it

  • Removing it entirely

  • Breaking it up (into need and capability)

  • Adding an intermediate task

  • Adding a new issue

These actions are available from the profile form of an entity:

  • Undoing the last action

  • Redoing the last undone action

  • Sending a message about it

  • Pasting a copied attachment to it

  • Adding a new issue
