IS plan = IS practices + environment

An IS plan can be understood as information sharing practices applied to an environment. Information sharing practices are definitions of tasks and the information flows between them. An environment is composed of agents (people and systems) in roles of responsibility within organizations. These agents are assigned to flow-connected tasks and consequently have information sharing commitments.

Figure 1.11. Flow-connected tasks in the IS practices model of an IS plan


In order to separate (decouple) IS practices and environment, it is recommended to model IS practices without mentioning actual organizations or actual people or systems. It is best to specify tasks as being executed by "disembodied" roles from types of organizations (e.g. a "forensic autopsy" is to be carried by a "coroner" from a "county health office"). IS practices models that are not attached to a particular environment can be applied, possibly with some local adaptations, to various environments.

Figure 1.12. Profile of an actual organization populating the environment of an IS plan


An IS plan's environment identifies the actual people or systems that play the organizational roles mentioned in the IS practices model (or IS model for short). For example, planners could build an IS model for responding to an epidemic outbreak at the state level, and then apply the same IS model to different states, each state being a different environment with its own organizations and role-assigned agents.

Note that there is not yet explicit support in the current release of Channels for separately managing the IS model and environment of an IS plan, or for explicitly composing IS plans by applying one reusable IS model to different environments. A forthcoming release of Channels will provide such support. In the meanwhile, planners are strongly encouraged to follow the above best practice of keeping the IS model as decoupled as possible from the IS plan's environment.
