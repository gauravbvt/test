Semantic proximity matching

Naming matters a lot in Channels. Entities are referenced by their unique names and are created when first named. Using different names for what is essentially the same role, type of organization etc. would become a source of confusion and would undermine the usefulness of an IS plan. Whether the names of tasks, flows, information needs, capabilities and information elements are different or the same determines whether needs are satisfied or satisfiable.

Channels uses semantic proximity matching to promote convergence between planners toward a consistent, shared vocabulary for a plan.

Semantic proximity matching works by looking for already used, similar names that could mean something close to the name currently being entered by a planner in a form. A drop down menu shows to the planner the alternate, semantically proximal names found. The planner may then elect to use a name already used instead of a new one with similar meaning.
