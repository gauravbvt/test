Connecting tasks with an information flow

A planner connects two tasks with an information flow to express the requirement that agents executing the source task commit to share information with agents executing the target task, possibly subject to some restrictions. The information is either produced by or acquired for the execution of the source task, and the information is to be used for the execution of the target task. The same flow appears to the source task in its list of "sends" and to the target task in its list of "receives".

A planner connects a source and target task by doing any of the following:

  • Opening an information capability of the source-to-be task and identifying or creating the target task in the "To task" field. The target task can be created in situ by selecting "Other..." in the list of candidate target tasks and naming a task that does not yet exist within the segment.

    If the flow is between tasks in the same segment, the information need is converted into an information flow; to preserve the information need, the planner must duplicate it and set the source on only the copy or the original. If the flow is between tasks in different segments, the information need is preserved when the flow is created.

  • Opening an information need of the target-to-be task and identifying or creating the source task in the "From task" field. The source task can be created in situ by selecting "Other..." in the list of candidate source tasks and naming a task that does not yet exist within the segment.

    If the flow is between tasks in the same segment, the information capability is converted into an information flow; to preserve the information capability, the planner must duplicate it and set the target on only the copy or the original. If the flow is between tasks in different segments, the information capability is preserved when the flow is created.

  • Selecting a task with at least one information need and executing the "Satisfy all needs" command on it. The task will be connected with all tasks, across all segments, that have matching information capabilities (if not connected already). The information need and all information capabilities are preserved.

Note that tasks in different segments can be connected by an information flow only if both tasks are both already defined and if an information need of one matches an information capability of the other (an unnamed need matches all capabilities and vice-versa).

Table 3.11. Scoping: Connect tasks with an information flow
Prerequisites:

  • The source or target task was added, or both if the tasks are in different segments.

  • An information need was added to the target task or an information capability was added to the source task, or both if the tasks are in different segments.
Steps:

(All steps are optional and they can be taken in any order.) Note: Tagging with information standard tags enables auto-population of standard elements of information.

  • Connect source task with target task (possibly creating either in the process). If connection is done by matching need and capability, the information flow thus created is initialized by reconciling the definitions of both need and capability. Note that if the flow crosses segments, restrictions are not reconciled and the new flow is not (yet) restricted.

  • Planners may want to modify the modify the resulting definition of the flow.
Outcomes:

  • The source task and target task are connected by an information flow from which information sharing commitments are computed.

  • Channels includes the flow in failure impact and information dissemination analysis.

  • Channels can check whether outgoing flows conform to the expectations (channels, timing, elements, etc.) expressed in the task's capabilities of the same name.

  • Channels can check whether incoming flows conform to the requirements (channels, timing, elements, etc.) expressed in the task's needs of the same name.

  • Channels can offer the task as a candidate source for the need of a task in the same segment.

  • Channels can offer the task as a candidate target for the capabilities of a task in the same segment.
