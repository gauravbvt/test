Acquiring and releasing edit locks

More than one planner can concurrently work on a plan. "Collisions" between two planners attempting to modify the same plan element are prevented by allowing only one planner at a time to modify any given task, flow, entity, segment definition, or plan definition.

When a planner opens an edit form on a plan element, Channels attempts to grab an edit lock on behalf of the planner for that plan element. If another planner already owns the lock, the form is shown to the first planner with the editable fields disabled.

A lock on a plan element owned by a planner is released when the planner collapses or closes its edit form, or logs out. An inactive planner eventually loses all acquired locks.

The name of the owner of a lock appears to other planners usually in the place of the "actions" menu for the locked plan element. A message can be sent to the owner of a lock by clicking on his/her name.

A planner waiting for edit locks is notified with a message in the segment bar when the locks are released. The planner is then instructed to refresh the editor in order to acquire the locks and enable disabled edit forms.
