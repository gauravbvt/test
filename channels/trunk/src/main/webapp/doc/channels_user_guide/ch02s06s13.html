Task overrides panel

A task can be defined in such a way as to be a more specific case of another task, in effect overriding it. An overridden task can not be assigned any of the agents assigned to the overriding task. This allows planners to model exceptions and special cases in an IS plan.

When a task is overridden or overrides another, planners can bring up an overrides panel for that task.
