Robustness issues

The IS plan as currently defined will generate IS procedures that are inherently failure-prone.

Cause

Severity

Remediation

Can be waived

Flow intended as an alarm is not a notification.

Depends on impacted tasks

  • Make the flow a notification.

Yes

No guarantee information will be received over broadcast channels.

Depends on impacted tasks

  • Add an alternate, nonbroadcast channel to the flow.

Yes

Tasks trigger each other in a loop

Low

  • Break the loop by having one of the triggered  tasks not trigger the other.

Yes

Received element of information is declassified when resent.

Medium

  • Set the classification of the element sent to be at least as high as when it is received, lower the classification of the received element, or do not send the classified element.

Yes

Sharing over undersecured channel

Medium

  • Remove the channel as an alternative, change the security profile of its medium, change the secrecy classification of the problematic elements of information, or remove the classified element(s) of information that exceed the channel's security.

Yes

Insufficient clearance

Medium

  • Declassify the information shared, increase the clearance of the agents assigned to the recipient task, or change the definition of the task to assign only cleared recipients.

Yes

Only one source of critical information

Depends on impacted tasks

  • Add an alternate source of make the information noncritical.

Yes

Potential task deadlock

Medium

  • Provide redundancy for at least one of these critical sharing flows.

Yes

Single point of failure

Medium

  • Generalize task specifications so that the agent is not the only one assigned to a critical task or add participating agents to the scope with the same role and organization.

Yes

Untimely necessary sharing

Depends on impacted tasks

  • Ease the time constraints the information need or reduce the time delay for the matching information sharing flow.

Yes

Untimely dissemination

Depends on impacted tasks

  • Ease the time constraints the information needs or reduce the time delays of flows disseminating needed elements.

Yes

Single agent assigned to critical task is bottleneck

Depends on impacted tasks

  • Change the specification of the task to allow more agents to be assigned to it or add participating agents to the scope who will be assigned to the task

Yes

Agents assigned to task receive information from too many sources

Depends on impacted tasks

  • Remove incoming sharing flows or add intermediates to spread the incoming communication load

Yes

Agents assigned to task have too many sharing commitments

Depends on impacted tasks

  • Remove outgoing sharing flows or add intermediates to spread the outgoing communication load

Yes

Important task has only one assignment

Depends on impacted tasks

  • Profile agents so that more than one match the task specifications or modify the task specifications so that it matches more than one agent.

Yes

Availability gap in task assignments

Depends on impacted tasks

  • Change the availabilities of assigned agents so that they leave no gap or change the specification of the task so that more agents are assigned to fill the gaps.

Yes

Important task without failure notification protocol

Depends on impacted tasks

  • Add a notification conditional on task failure

Yes

Important task is not operational.

Depends on impacted tasks

  • Make the task operational

Yes

Important sharing flow is not operational.

Depends on impacted tasks

  • Make the flow operational

Yes

Ambiguous information sharing

Depends on impacted tasks

  • Require that the information sharing reference the event context or rename the sharing flow.

Yes
