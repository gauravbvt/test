Place profile panel

When profiling an actual place, a planner can identify what other place, if any, contains it. A planner can also give the place a street address, postal code, and a geoname (the name of a town, state, mountain, lake etc.). If the geoname is ambiguous, more specific geonames are offered for confirmation.

Figure 2.111. Place profile panel - Details view


Figure 2.112. Place profile panel - Details view (cont.)


When profiling a type of place, a planner can specify that actual places of this type must:

  • Be inside

    • the plan's locale

    • the location of an event in the plan's scope

    • or another actual or type of place

  • Contain

    • the plan's locale

    • the location of an event in the plan's scope

    • or another actual or type of place

Figure 2.113. Place type profile panel - Details view


There is of course a Map view for actual place entities, and an Issues view for both actual places and place types.
