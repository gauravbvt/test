Changing the categorization of an entity

Entity types can themselves be categorized by other entity types. For example, a "Rail station" (a place entity type) can be categorized as a "Building" (another place entity type) and as a "Transit system node" (another place entity type). The categorization of entity types together creates typologies. Because tasks can be defined in terms of entity types (agent types, roles, place types, organization types), small changes to entity typologies can have wide-ranging effects on task assignments, and thus information sharing commitments.

It is highly recommended that planners frequently look up typologies (agent typology, event typology, role typology, etc.) to make sure that each represents an intended conceptualization of the plan's environment and check the assignments calculated for tasks that are affected by the change.
