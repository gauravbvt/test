Information sharing model page

The information sharing model page is also known as the plan editor. It opens on the version (development or production) of a plan selected in the home page. A user can only select plans for which he/she has planner privileges (unless the plan version is marked as a template).

Only the development version of a plan can be modified. A production version is “read-only”. In a read-only version, all form fields are disabled and no survey can be launched about issue resolution.

When the plan editor opens up on (a version of) a plan, Channels pre-selects a segment and a task within it. A segment and a task are always selected. The task with its “receive” and “send” information flows are shown “collapsed” and summarized.

If the plan is brand new, an empty task in the otherwise empty and untitled segment is selected. The last task in a segment can not be removed. The last segment in a plan can not be removed.

A planner can at any time switch segment, if more than one is defined. The tasks and information flows of the selected segment are visualized as a flow map (a directed graph). The selected task is highlighted (by a gray outlined rectangle) in the flow map.

The Plan editor window has four principal components:

  • The segment bar at the top

  • The information flow map

  • The panels showing the selected task with its receives and sends at the bottom

  • The collaboration tab panels to the right

The segment bar

The segment bar identifies the currently selected segment within the plan. If applicable, a drop down menu gives the planner access to others segments within the plan.

Figure 2.22.  The segment bar


The Help icon opens up this content for browsing in a separate window.

The Left and right arrow buttons are Channels' "back and forward" buttons. The back button brings the plan editor to its previous state, that is, re-opening whatever panel was just closed or collapsed, showing the previous segment that was just switched from etc. The forward button brings back the state the planner backed from.

The plan editor uses AJAX to provide a user experience similar to that of a "native" application (changes to the content without navigating to another page or redisplaying the page). The "back and forward" buttons must be used instead of the browser's back and forward button which would navigate users out of the plan editor.

Show menu

The segment bar's "Show" menu gives access to:

  • the collaboration tab panels, if collapsed

  • the plan details panel

  • the segment details panel - Opens the segment details panel. Clicking on the name of the segment is a short cut.

  • the surveys panel

  • a map of all segments in the plan

  • all issues in the plan

  • the entity typologies panel

  • the plan index

  • Channels help (this content)

Figure 2.23. Main "show" menu


Actions menu

The segment bar's "Action" menu enables the planner to act on the segment as a whole by:

  • sending a message about the segment

  • adding a new task

  • adding or pasting a copied task

  • adding an "ad hoc" issue to the segment

  • pasting a copied attachment to the segment

  • moving tasks from this segment to another

Other available actions include:

  • Undoing the last action (if possible)

  • Redoing the last undone action (if possible)

  • Removing the current segment from the plan

  • Logging out of Channels
