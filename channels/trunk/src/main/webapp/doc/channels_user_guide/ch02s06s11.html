Entity profile panels

A new entity (an agent, role, organization, place, event, phase, or transmission medium) is automatically created when it is first referenced by name. Channels requires that all entities of a given kind be assigned unique names (John Smith, John Smith (1), John Smith (2), etc.).

The definitions of plan elements can reference both actual entities (ACME Corporation, Peter Sanchez, Morristown NJ) and types of entities (Corporation, Certified Emergency Technician, City). An entity is categorized by listing the names of entity types. An entity can only be categorized with entity types of its own kind (Warehouse A can be categorized as a building but not as a senior executive). The profiles of actual entities and entity types can be edited.

A profile panel for an entity, actual or type, can be opened from wherever the entity is referenced by name, be it in the specification of a task, in the definition of a segment, in an index, etc.

As a rule, entity profile panels have these views:

  • A Details view where entity attributes are edited

  • A Network view, where applicable, showing how the entity relates to other entities of the same kind

  • A Map view, where applicable, showing in a separate window where the entity is located

  • An Issues view where issues that relate to the entity are collected

An entity profile panel opens by default in the Details view. The Details view contains a form for editing the attributes of an entity. All entity profiles, whether they are actual or types, are defined by:

  • A unique name

  • A description (free text)

  • Tags

  • A categorization (a list of entity types of the same kind)

  • Document attachments

In addition, an entity profile includes:

  • Some analytical content that relates to the entity

  • An index of all references to the entity, and matches with the entity, if the entity is a type

  • A list of all issues about the entity itself or that relate to the entity

  • A filterable collection of all other entities that relate to the entity

Each kind of entity has additional, specific attributes and these are reflected in the Details view for each kind of entity.

The Network view in the entity profile panel is available for agents, roles, and organizations. It shows how such entities have information sharing commitments with others of the same kind.

If the profiled entity is geo-locatable, a Map view shows where it is located in a different window. A Map view is available for organizations, events, and, of course, places.

An Issues view can show not only issues about the entity, but issues that relate to it. It is available for all kinds of entities.

The Actions menu in profile panels allows planners to:

  • Undo the planner's last change, if possible

  • Redo the last undone change, if possible

  • Send a message about the profiled entity

  • Add an attachment copied from another plan element, if any

  • Add an ad hoc issue to the profiled entity

Role profile panel

Roles are always entity types, so there's only one kind of profile panel for roles. The edit form in the Details view is a simple one because the profile of a role only has those attributes that are common to all entities.

Figure 2.91. Role profile panel - Details view


Figure 2.92. Role profile panel - Details view - analytical content expanded


The Network view shows how agents, aggregated in the profiled role, have sharing commitments with agents in other roles, also aggregated. The role-playing agents and their sharing commitments aggregated in the visualization are separated out in the sortable, filterable, pageable table underneath the diagram.

The Issues view collects issues about or related to the role and presents them in a filterable, sortable, pageable table. Planners can control which issues are displayed.
