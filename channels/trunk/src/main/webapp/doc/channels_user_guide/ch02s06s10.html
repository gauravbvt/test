Segment definition panel

Planners can edit the scope, name and description of a segment. A segment's definition panel is accessed by selecting “About plan segment” from the “Show” menu in the segment bar, or by clicking on the name of the segment also in the segment bar.

Figure 2.82. Accessing a segment's definition panel


This opens the Details view in the segment definition panel. There, the segment can be renamed, and the phase and event it references can be changed as well as the event context, if any. The event can be given the minimum rating (low, medium, high, or highest) at which the tasks and flows in the segment apply. As with any other kind of plan element, reference and policy documents can be attached to a segment, and ad hoc issues can be added.

Figure 2.83. Segment definition panel - Default view


The "Show" menu in the segment definition panel gives access to views other than the default Details view, namely:

  • Goals view - to view and edit the goals of the segment

  • Organizations view - to view the organizations involved in this segment and the sharing commitments between them

  • Map view - to locate segment tasks on a Google map

Figure 2.84. Segment definition panel - Show menu


The segment is further defined by identifying and describing a set of goals that relate to the event phase addressed by the segment. Goals can be risks to be mitigated or gains to be made. These goals are expected to be realized by tasks in the segment. A segment is expected to have at least one goal.

The Goals view of a segment lets planners define the goals of organizations that are in scope for this segment. Goals can be added and removed. The goal of an organization would be to mitigate a risk or to make a gain. A goal is defined in terms of:

  • A type picked from a taxonomy of risks or gains, an importance (minor, major, severe, or extreme for risks; low, medium, high, or very high for gains)

  • The identity of the organization (either an actual organization or a type of organization)

  • Whether the goal goes away when the event phase covered by the segment terminates

The tasks that mitigate a given risk or achieve a goal in scope can be listed, sorted and navigated to.

Figure 2.85. Segment definition panel - Goals


The Map view opens a window with a Google map showing all tasks in the segment that specify geo-locatable locations. The geomapping of elements is available whenever applicable throughout Channels.

Figure 2.86.  Google map of located tasks in a segment


The Actions menu of the segment definition panel enables the planner to undo/redo past changes, send messages about the segment, and add issues about the segment as a whole.

Figure 2.87. Segment definition panel - Actions menu
