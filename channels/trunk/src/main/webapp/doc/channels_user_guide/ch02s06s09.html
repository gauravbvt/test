Plan definition panel

The plan definition panel is opened by selecting the item “About Plan” in the top “Show” menu.

Figure 2.63. Access to plan definition panel


Planners use the plan definition panel's details view to change the name of a plan, give it a description, define phases to be applied to events, localize the plan, attach documents to it, and add ad hoc issues about the plan as a whole.

Figure 2.64. Plan definition panel - base view


The "show" menu of the plan definition panel gives access to other views on the plan, namely:

  • A list of all events referenced in the plan, with the ability to declare certain events as "incidents" (i.e. "outside" events not expected to be caused by planned tasks)

  • The list of all secrecy classifications (also referenced in the plan), with the ability to add and edit secrecy classifications,

  • The list of all organizations in scope, where a planner can indicate if an organization is expected to be assigned planned tasks,

  • A visualization and analysis of how all segments inter-relate

  • A visual exploration tool of assignments and commitments (IS procedures) computed from the plan

  • A searchable who's who of all participating agents

  • A searchable list of all issues anywhere in the plan

  • A searchable list of all document attachments

  • An index of all named plan elements (tasks, flows, entities etc.)

  • An overall evaluation of the plan with issues metrics

  • A list of all registered users and their identification with agents in the plan (participations)

  • A mechanism for voting to release the current development version of the plan into production

Figure 2.65. Plan definition panel - show menu


In the All events view, a planner can add events to the scope of the plan. A planner can mark an event as an incident if it is caused “outside” of the plan and not as an outcome of a planned task. There is always at least one event defined in a plan. Un-marking as incident an event that is not defined and is not referenced anywhere in the plan removes it from the plan.

Figure 2.66. Plan definition panel - all events


In the Secrecy classification view, a planner can view all defined secrecy classifications that can be used to qualify elements of information (classification), transmission media (security) and agents (security clearances). Classifications can be re-ordered, references to a classification can be listed, unreferenced classifications can be removed and new classification systems and classifications added.

Figure 2.67. Plan definition panel - secrecy classifications


In the All organizations view, a planner can see all organizations within the scope of the IS plan and add new ones with expectations that they be assigned tasks. An organization in scope that is expected to have tasks but is assigned none raises a completeness issue.

Figure 2.68. Plan definition panel - all organizations - scope


A network view of all inter-organization sharing commitments is available to explore the sharing commitments implicit in a plan that cross organizational boundaries.

Figure 2.69. Plan definition panel - all organizations - network


In the All segments view, planners can look at a diagram of all segments in the plan. The segments can be grouped by phase or by event. Planners can browse all cross-segment sharing flows as well as all tasks in one segment that start or terminate (the event phase of) another segment.

Figure 2.70. Plan definition panel - all segments


The procedure map view lets planners visualize and explore the assignments and commitments implicit in the flow maps. Remember that task assignments and information sharing commitments are implicit in a plan and are computed by Channels by matching known agents with task assignee specifications, and by applying the sharing restrictions set on flows that connect tasks to which agents are assigned. In other words, assignments and commitments (IS procedures) are the result of applying an environment (known agents employed by known organizations) to an informations sharing model (specifications of tasks and flows).

Because full procedure maps can quickly become quite large and unwieldy, the procedures map view allow planner to manage this complexity by:

  • focusing on the assignments and commitments of (actual) agent or organization (as an aggregate of employed agents)

  • looking at the procedures from a single segment or the entire plan

  • aggregating the assignments of those who commit to share with the agent or organization in focus by

    • type of organization

    • type of organization and role

    • organization (actual)

    • organization and role

Figure 2.71. Plan definition panel - procedures map


In the Who's who view, planners can search a list of all agents known to the plan, and see their responsibilities.

Figure 2.72. Plan definition panel - who's who


In the All issues view, planners can sort, filter and browse all issues currently detected in or manually added to the plan. Surveys on individual issues can be created or accessed from this view.

Figure 2.73. Plan definition panel - all issues


In the Bibliography view, planners browse and can access all the documents that are attached to plan elements.

Figure 2.74. Plan definition panel- bibliography


In the Index view, planners can find, filter, sort and navigate to all named plan elements.

Figure 2.75. Plan definition panel - index


Entities can be categorized by entity types of their kind (agents are categorized by agent types, role are categorized by other roles etc.). The All types view shows planners the typologies implied by the categorizations of entities.

Figure 2.76. Plan definition panel - all types


The tags defined and used in a plan are consolidated in the All tags view where a planner can find what tags are used in a plan and what elements are tagged by a given tag.

Figure 2.77. Plan definition panel - all tags


The Evaluation view gives a summary evaluation of the plan's fitness by providing metrics about validity, completeness and robustness issues.

Figure 2.78. Plan definition panel - evaluation


The Participations view lists all registered users and the identity of the agent, if any, that is associated with each user. The planner can make or change these associations as well as update the contact information a participant inherits from the associated agent. A user who is not associated with an agent does not have information sharing guidelines.

Figure 2.79. Plan definition panel - participations


The Versions view is where planners vote to move the development version of the plan into production. Note that only an administrator can actually move a development version into production from within the Administrator page.

Figure 2.80. Plan definition panel - versions


The actions menu of the plan definition panel enables the planner to undo/redo past changes, send messages about the current plan version and add issues about the plan as a whole.

Figure 2.81. Plan definition panel - actions menu
