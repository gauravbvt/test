Profiling an agent

Agents are people or systems who, as responsible members of organizations, are assigned to tasks and commit to share information with other agents. An actual agent can play more than one role in more than one organization.

An agent type represents a category of agents that is distinct from roles. Agent types are often used to represent a qualification such as "Certified hazmat technician" or a social status such as "United States Citizen". An actual agent's security clearances, if any, are examined when the agent is the beneficiary of sharing commitments for classified information. An agent's contact information is shown in IS guidelines.

Table 3.17. Profiling: Profile an agent
Prerequisites:

  • The agent was named
Steps:

(All steps are optional and they can be taken in any order)

  • Change the name of the agent

  • Describe the agent

  • Tag the agent

  • Categorize the agent with other agents (avoiding circular categorization)

  • Attach picture, policy, or reference document

  • Give the agent's availability

    • as always available

    • or as a list of day-to-day time periods when available

If the agent is actual:

  • Set the agent as a system, or not

  • Set the agent as an archetype, a placeholder, or neither

    • To switch between archetype and placeholder, the current setting must first be unchecked

    • If set to placeholder, indicate if multiple users can be associated or only one

  • Give the agent's contact information as a list of communication channels (channel = transmission medium + "endpoint")

  • Give the agent's security clearances
Outcomes:

  • Depending on its categorization, the agent type may or may not be compatible as a category for certain other roles.

  • The tasks assigned to the agents and consequent information sharing commitments may change following a change in its categorization.
