From Plans to Procedures

A Plan consists of tasks that achieve organizational goals during various phases of anticipated events. These tasks consume information that may be produced by other tasks and then shared by the agents who execute them.

Known agents from participating organizations are assigned (or not) to tasks based on whether the profile of each agent matches the specifications of who is meant to execute a given task. The profile of an agent is defined in terms of the agent’s qualifications, the roles the agent plays in organizations, and the agent’s jurisdictions.

When a flow connects an information-producing task (source task) with an information-consuming task (target task), then all agents assigned to the source task, possibly subject to restrictions, commit to sharing information with the agents assigned to the target task. The information is strictly to be used for the execution of the target task.

The sum of all task assignments and sharing commitments for a given agent is this agent’s Procedures.
