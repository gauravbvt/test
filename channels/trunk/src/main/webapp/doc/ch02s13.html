Plan definition and analytics

A planner selects the item “About Plan” in the top “Show” menu to access views about the entire Plan.

The default Plan view is theDetails view. Here, planners can change the name of a Plan, add phases and remove unreferenced phases. Planners can also add or view issues about the Plan as a whole.

In theAll events view, a planner can add events to the scope of the Plan. A planner can mark an event as an incident if it is caused “outside” of the Plan and not (always) as an outcome of a planned task. There is at least one event defined in a Plan.

In theSecrecy classification view, a planner can define secrecy classifications that are then used to qualify elements of information and to set individual clearances. References to a classification can be listed. Unreferenced classifications can be removed.

In theAll organizations view, a planner can identify the organizations that are in the default context of the Plan.

In theAll segments view, planners can look at a map of all segments in the Plan. The segments can be grouped by phase or by event. Planners can browse all cross-segment sharing flows as well as all tasks in one segment that start or terminate another segment.

In theWho's who view, planners can browse a directory of all agents involved in the Plan.

In theAll issues view, planners can sort, filter and browse all issues currently detected or manually added within the Plan. Surveys on individual issues can be created or accessed from this view.

In theBibliography view, planners browse all the documents that are attached to Plan elements anywhere in the Plan.

In theIndex view, planners can find, filter, sort and navigate to all entities referenced in the Plan.

The Evaluation view gives a summary evaluation of the Plan in terms of categories of issues detected and added by planners.

Finally, the Version view is where planners vote to move the development version of the Plan into production. A unanimous vote is required. Moving a development version into production creates a development version from a copy of the new production version. Planners can be contacted by email from within the view to prompt them to vote.
