ISP issues

Issues relate either to the validity, completeness or robustness of a Plan. The detection of issues can be automated by software.

Validity issues

The value given to an attribute of a Plan or one its elements is either inherently wrong or it causes a logical inconsistency.

Severity

Remediation

Medium

or, in either case,

  • use another place as the plan's locale

  • or leave the plan's locale unnamed.

Completeness issues

An element of the Plan is clearly missing given the scope of the Plan and the other elements it contains.

Cause

Severity

Remediation

Can be waived

Agent not member of any known organization

Low

Make the agent the member of an organization.

No

Sharing agreement encompassed by another

Low

Remove either sharing agreement.

Yes

Sharing agreement does not match any sharing commitment

Low

  • Remove the sharing agreement

  • or add flows that imply a commitment covered by the agreement.

Yes

Immediate or permanent task is unnecessarily triggered

Low

  • Have no flow trigger this task

  • or have the task not start with the plan segment.

Yes

Organization expected to have task assignments but has none

Low

  • Remove the expectation on the organization

  • or add a task that will be assigned to the organization or one of its sub-organizations, if any

No

Organization is missing expected commitment

Low

Fulfill expectation by having a task of a given category and assigned to an organization connected to another task of an expected category and assigned to an organization with the appropriate relationship, with a flow with an expected intent. For example, an audit task expects to receive a report from an operations task assigned to a child organization.

Yes

Organization is not assigned a category of tasks

Low

Specify at least one task of the missing category that is assigned to the organization or one of its sub-organizations.

Yes

Report flow shares no elements of information

Low

  • Specify elements of information

  • or change the intent to something other than report

No

Sharing commitment by an organization without required agreement

Low

  • Confirm an agreement covering this sharing commitment

  • or remove the requirement for agreements for the organization.

Yes

  • Make sure that the agent has a known endpoint (address, phone number etc.) for contact via the unicast channel

  • or remove the unicast channel from the flow.

No

Planned-for event may never be caused

Medium

No

Segment’s event phase may never end

Medium

If the segment is for a co-event phase,

  • have the event end on its own

or if the plan segment is for a pre-event phase

  • have the event be an incident

  • or have a task in any segment cause it

or, in all cases,

  • have at least one task in the segment terminate the event phase.

Yes

Segment's event phase may never start

Medium

If the segment is for a co-event phase,

  • have the event be an incident

  • or have a task in another segment cause it

or if the plan segment is for a post-event phase

  • have the event be self-terminating

  • or have a task in a co-event segment terminate its segment.

Yes

Sharing flow implies no commitments

Medium

  • Modify the specifications for source and target tasks so that they both imply assignments

  • or add or redefine participating agents to match source and target task specifications

or, if inadequate clearances are at fault,

  • declassify elements of information

  • or make sure otherwise committed agents have the required clearances.

Yes

Task is triggered but never started

  • else, make that the triggering task itself can start.

No

Robustness issues

The Plan as currently defined will generate Procedures that are inherently failure-prone.

Cause

Severity

Remediation

Can be waived

Flow intended as an alarm is not a notification

Depends on impacted tasks

Make the flow a notification

Yes

No guarantee information will be received over broadcast channels.

Depends on impacted tasks

Add an alternate, non-broadcast channel to the flow

Yes

Tasks trigger each other in a loop

Low

Break the loop by having one of the triggered  tasks not trigger the other.

Yes

Received element of information is declassified when resent

Medium

  • Set the classification of the element sent to be at least as high as when it is received

  • or lower the classification of the received element

  • or do not send the classified element

Yes

Sharing over under-secured channel

Medium

  • Remove the channel as an alternative

  • or change the security profile of its medium

  • or change the secrecy classification of the problematic elements of information

  • or remove the classified elements(s) of information that exceed the channel's security

Yes

Insufficient clearance

Medium

  • Declassify the information shared

  • or increase the clearance of the agents assigned to the recipient task

  • or change the definition of the task to assign only cleared recipients

Yes

Only one source of critical information

Depends on impacted tasks

  • Add an alternate source

  • or make the information non-critical

Yes

Potential task deadlock

Medium

Provide redundancy for at least one of these critical sharing flows

Yes

Single point of failure

Medium

  • Generalize task specifications so that the agent is not the only one assigned to a critical task

  • or add participating agents to the scope with the same role and organization

Yes

Untimely critical commitment

Depends on impacted tasks

  • Ease the time constraints the critical need

  • or reduce the time delay for the matching information sharing flow

Yes

Task-triggering notification is not timely

Depends on impacted tasks

  • Ease the time constraints the triggering need

  • or reduce the time delay for the matching notifying sharing flow

Yes

Single agent assigned to critical task is bottleneck

Depends on impacted tasks

  • Change the specification of the task to allow more agents to be assigned to it

  • or add participating agents to the scope who will be assigned to the task

Yes

Agents assigned to task receive information from too many sources

Depends on impacted tasks

  • Remove incoming sharing flows

  • or add intermediates to spread the incoming communication load

Yes

Agents assigned to task have too many sharing commitments

Depends on impacted tasks

  • Remove outgoing sharing flows

  • or add intermediates to spread the outgoing communication load

Yes
