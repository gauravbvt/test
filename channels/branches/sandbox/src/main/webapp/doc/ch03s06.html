Entities

A new entity (an agent, role, organization, place, event, phase or transmission medium) is automatically created when it is first referenced by name. Channels requires that all entities of a given kind be assigned unique names (John Smith, John Smith (1), John Smith (2) etc.).

Recall that the definitions of Plan elements can reference both actual entities (ACME Corporation, Peter Sanchez, Morristown NJ) and types of entities (a corporation, a certified emergency technician, a city). An entity is classified by tagging it with the names of entity types. An entity can only be tagged with entity types of its own kind (Warehouse A can be tagged as a building but not as a senior executive). Both actual entities and entity types can be edited.

An editable profile of an entity , actual or type, can be opened from wherever the entity is referenced by name, be it in the specification of a task, the definition of a segment, or in an aggregate or analytical view.

Keep in mind that events, roles and transmission media in a Plan are always types. There is no actual event in a Plan because events are always anticipated. There is no such thing as an “actual role” because roles are always concepts. Transmission media are also always types because a Plan only cares about the class of a transmission medium.

The Details view shows the profile of an entity. All entity profiles, whether they are actual or types are composed of:

  • a unique name

  • a description (free text)

  • a classification (a list of entity types of the same kind)

  • document attachments

In addition, an entity profile always includes an index of all references to the entity (and matches, if the entity is a type), a list of all issues about the entity proper, and a filterable collection of all entities that releate to the entity.

Each kind of entity has additional, specific attributes that can also be set.

Profiling an organization

A planner profiling an actual organization can define its mission and set its location, contact information, and parent organization.

A planner can declare that it si an issue if a sharing commitment by the organization with another is not covered by an agreement, and that it is also an issue if there is a role assigned by the organization in the Plan for which no agent is known.

A planner can add entries to an organization's membership/employment roster, by naming for each entry an agent, a role, a jurisdiction (optional), a title and a supervisor agent (optional). Alternately, the planner can confirm memberships that are already implied by task specifications (a task specification can identify an agent in a role within a jurisdiction for an organization).

Inter-organizational sharing commitments are inferred from information flows that are sourced from tasks assignable to members of the organization. A sharing agreement may or may not exist for a sharing commitment that crosses organizational boundaries. It is an issue if an inter-organizational sharing commitment is unconfirmed as required by the organization. When viewing an organization, a planner can see the sharing commitments the organization makes with other organizations and confirm whether each commitment is covered or not by an agreement.

In the Details view of any kind of entity, references to the entity are listed in a filterable and paged index. Task assignments and sharing commitments for the organization, as well as issues about the organization, if any, are shown at the bottom of the view.

Profiling an agent

The profile attributes specific to an actual agent are: whether or not the agent is an archetype, whether it is a system or a person, its contact information (transmission media with endpoints where applicable), and its security clearances (secrecy classifications).

  • whether or not the agent is an archetype,

  • whether it is a system or a person,

  • its contact information (transmission media with endpoints where applicable),

  • and its security clearances (secrecy classifications)

In addition to profile attributes, the Details view of an agent lists the roles played by the agent, the agent's task assignments and sharing commitments, all references to the agent (indexed) and all issues detected or reported specifically about the agent.

Profiling a place

When profiling an actual place, a planner can identify what other place, if any, contains it. A planner can also give the place a street address, postal code and a geoname. If the geoname is ambiguous, more specific geonames are offered for selection.

When profiling a type of place, a planner can specify that for actual places to be of this type they must:

  • must be inside

    • the plan's locale or

    • the location of a planned event or

    • another actual or type of place

  • must contain

    • the plan's locale or

    • the location of a planned event or

    • another actual or type of place

Profiling a transmission medium

When editing a transmission medium, a planner can set the transmission mode to either unicast (one -to-one), multicast (one-to-many) or broadcast (one-to-any), unless the mode is already implied by another medium this one is tagged with (to make it a special case of it).

The address pattern is a “regular expression” that constrains the form an “address” can take for this medium. An empty pattern imposes no constraint.

A planner can identify the media, if any, this one can interface with as it carries out transmissions from sender to receiver.

Finally, a planner can describe the security of the transmission medium in terms of security classifications. Qualifying a medium as “US Gov/Top Secret” means that Top Secret information can be transmitted securely over this medium.

Keep in mind that all transmission media are types; there is no such thing in ISP as an actual transmission medium.

Defining an event

An event in ISP is always an entity type (all events in ISP are anticipated). A planner can set the location (actual or type) where the event would occur. A planner can also indicate whether or not this type event can terminate on its own, without the intervention of any agent.

Defining a phase

Planners can introduce any number of phases within the scope of a Plan. In addition to setting generic entity attributes (name, description, classification), a planner indicates whether a phase is before an event (pre-event), during an event (co-event) or after an event (post-event).

Entity network views

The Network view is available on agents, roles, and organizations. It displays a network graph depicting how the entity is connected by information flows to other entities of the same kind via sharing commitments. This analytical view allows a planner to filter commitment-generating information flows to, say, show only the ones from tasks executed by a specific role. The network graph is navigable. The list of flows can be further filtered, sorted and paged. Listed flows can also be navigated to.

Entity issues view

The Issues view collects and filters issues that are about an entity or that are related to it.

Waived issues can be included in or excluded from the sortable, page-able list of issues.

Surveys, whether ongoing, closed, or to-be-created, are accessible from each listed issue. Only planners can launch or close surveys. A survey gathers the opinions of planners and non-planners alike on how a particular issue should be remediated.
