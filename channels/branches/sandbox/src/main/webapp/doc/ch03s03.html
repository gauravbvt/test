Plan editor panels

The Plan editor opens on the development version of one of the Plans for which the user has planning privileges.

When the Plan Editor opens up on a Plan, Channels pre-selects a segment and a task within it. The task with its “receive” and “send” information flows are shown “collapsed” and summarized.

A planner can at any time select to see another version of the Plan, if any. A planner can also switch to another assigned Plan. Only the development version of a Plan can be modified; retired and production versions are “read-only” (all form fields are disabled, no survey can be launched).

A planner can at any time select which segment within a Plan to view or work on. The tasks and information flows of the selected segment are visualized as a flow map (a directed graph). The selected task is highlighted (by a gray rectangle) in the flow map.

The Plan editor window has four principal panels:

  • The Plan and Segment bar at the top

  • The information flow map

  • The panels showing the selected task at the bottom

  • The collaboration panel to the right

The Plan and Segment Bar

The Plan and Segment bar identifies the currently selected segment within the currently edited Plan. If applicable, drop down menus give the planner access to other versions of the Plan, to other Plans, and to others segments within the current Plan.

A "Show" menu gives access to:

  • the collaboration panel, if collapsed

  • the Plan panel

  • the Segment panel

  • surveys

  • all generated Procedures

  • the administration panel (if the use has Administrator privileges)

  • a map of all segments in the Plan

  • and a help window

An "Action" menu enables the planner to act on the segment as a whole by:

  • sending a message about the segment

  • adding or pasting a copied task

  • adding an "ad hoc" issue to the segment

  • pasting a copied attachment to the segment

  • moving tasks from this segment to another

Other available actions include:

  • Undoing the last action (if possible)

  • Redoing the last undone action (if possible)

  • Removing the current segment from the Plan

  • Logging out of Channels

The Help icon opens a Help window with this content.

The Left and righ arrow buttons are Channels' "back and forward" buttons. They are to be used instead of the browser's back and forward button which take users out of Channels.

Clicking on the name of the segment (upper left corner) opens up the "About segment" panel where the name, description and definition of the segment can be edited.

Information flow map

Tasks are the nodes (icons over text) in the graph and the information needs, capabilities and sharing flows are the arrows.

Tasks and flows are drawn in red if they have known issues. Green and red traffic lights represent the start and end of the event phase covered by the segment. Tasks that are triggered by the start of the segment are connected to the green traffic light icon. Tasks that terminate the event phase are linked to the red traffic light.

Information flows (information shared), information needs (information needed for carrying out a task) and information capabilities (shareable information produced by carrying out a task) are shown as arrows. Single-headed arrows represent notifications. Double-headed arrows are requests, with the empty arrow head pointing to the task that consumes the information request, and with the full arrow head pointing to the task that consumes the answer (the requested information flows to it).

Small squares indicate to-be-determined tasks. An arrow from a task to a dot represents an information capability of the task (information produced by the task that can be shared for use in other tasks). An arrow from a small square to a task represents an information need of the task (the task needs information possibly produced by other tasks).

When risks and goals are shown in a flow map they appear respectively as colored flags and stars connected to tasks and segment start and end icons.

Tasks within a blue rectangle denote tasks from other segments that are connected by sharing flows to tasks in the selected segment.

A flow map legend can be shown by clicking on the "help" icon in a flow map.

Task and flow panels

The task and flow panels at the bottom of the Plan editor show summaries of the selected task (there is always a task selected) together with a list of summaries of its "receives"( the task's information needs and information flows to it) and a list of its "sends" (the task's information capabilities and information flows from it). Each summary can be expanded into a form.

Each of the summaries is prefixed with a colored bullet. The color of the bullet indicates how important a task or a flow is. The color indicates the calculated impact of the task or the flow failing completely. Blue indicates a minor impact, yellow is major, orange is severe and red is extreme.

Collaboration panel

The collapsible "collaboration" panel on the right side lets a planner see who else is working on the Plan and what they have done (their activities). A planner can send messages to a planner or to all and can read received messages. The collaboration panel can be collapsed if more screen space for the flow map and task/flow forms is needed.

Display controls

The Plan editor gives the planner some control over the display of flow maps, forms used to specify tasks, flows, entities etc., and of the collaboration panel.

Flow maps can be quite large. A flow map can reduced to fit within the available space by clicking on the zoom-out icon. Clicking on the icon once more brings the flow map back to its normal size.

A flow map can be maximized to use all of the screen space allocated to the Plan Editor by clicking on the "maximize" icon. A maximized flow map can be reduced to fit. A flow map is un-maximized either by clicking once more on the "maximize" icon or by selecting a task or a flow within the map.

A flow map, by defaut, does not display risks and gains (goals) and how they relate to tasks or the segment as a whole. To display risks and gains, the planner must click on the "goals" icon. Clicking again hides the goals.

Clicking on the summary of a task or flow will open its form. Channels automatically adapts the width of task and flow forms to ease editing. Clicking o the summary closes the form.

The planner can expand the height of the forms, at the expense of the flow map, by pressing the "expand forms" icon.
