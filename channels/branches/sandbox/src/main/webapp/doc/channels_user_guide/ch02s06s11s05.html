Transmission medium profile panel

When editing a transmission medium, a planner can set the transmission mode to either unicast (one -to-one), multicast (one-to-many) or broadcast (one-to-any), unless the mode of this medium is already implied by that of another medium categorizing it.

The address pattern is a “regular expression” that constrains the form an “address” can take for this medium (see http://www.regular- expressions.info/ for a tutorial on regular expressions). An empty pattern imposes no constraint on the value of an address/endpoint.

A planner can identify what media this one can interface with (delegates to) as it carries out transmissions from sender to receiver. For example, a notification system would interface with phone, email, pager etc.

Finally, a planner can describe the security of the transmission medium in terms of secrecy classifications. Qualifying a medium as “US Gov/Top Secret” means that Top Secret information can be transmitted securely over this medium.

Keep in mind that all transmission media are types; there is no such thing in ISP as an "actual" transmission medium.
