Tagging

Tags provide a useful mechanism for the ad hoc cataloguing of elements of an IS plan that otherwise may have nothing else in common. Most elements of a plan can be tagged. An element can have multiple, comma-separated tags.

Figure 1.41. A tagged information flow


A planner can see all the tags used in an IS plan look up all elements of the plan tagged with a given tag.

Figure 1.42. Indexing by tags


The set of all tags in an IS plan is called the tag domain. Tags can be grouped by named sub-domains. When a tag belongs to a sub-domain, the tag is written as the name of the sub-domain followed by a colon followed by the name of the tag. For example, "esf:transportation" is the tag "transportation" in the tag sub-domain "esf". A planner can easily look up all elements tagged with tags belonging to a given sub-domain. Tag sub-domains can be pre-defined and attached to an IS plan (an attachment of type "Tag") prior to the tags being used. Note that tags need not be pre-defined to be used in tagging plan elements.

Figure 1.43. Files defining tag sub-domains attached to an IS plan


A special kind of tag sub-domain is called an "information standard". Each information standard tag is associated with a list of pre-defined elements of information, each with a name, and optional description and handling instructions. When a flow is tagged with an information standard tag, its elements of information can be auto-populated with those associated with the tag. An information standard tag sub-domain is added to an IS plan by attaching a file defining them (an attachment of type "Info standards").

Figure 1.44. Tagging a flow with an info standard tag to auto-populate its elements of information


Documents defining basic or info standard tag sub-domains must adhere to simple formatting rules. For basic tag sub-domains, the rules are:

  • A blank line or line starting with # is ignored.

  • Each line contains a single fully named tag (e.g. esf:transportation) or comma-separated tags (e.g. esf:transportation, esf:hazmat).

Figure 1.45. File defining basic tags for the "esf" tagging sub-domain


For info standard tag sub-domain files, the rules are:

  • A blank line or line starting with # is ignored.

  • A tag name is given in full and starts at the beginning of a line.

  • The elements of information for a tag each appear indented on a separate line beneath the tag.

  • The optional description and handling instructions are separated each by a hyphen.
