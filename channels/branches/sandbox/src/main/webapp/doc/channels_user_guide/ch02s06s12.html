Task assignments and sharing commitments panels

Recall that a task can identity one assignee explicitly as an actual agents but that tasks assignees are usually specified in terms of types of agents, roles, jurisdictions (actual or types of places), and organizations (types or actual). A task thus defined could be assigned to more than one agent. Channels looks up all agents known to a plan to determine which ones are to be assigned to the task.

A planner can look up the list of known agents that are implicitly or explicitly assigned to the task (the task assignments).

Agents are implicitly bound by sharing commitments when they are assigned to tasks that are sources of shared information, subject to sharing restrictions. All agents assigned to the "source" task have sharing commitments with all agents assigned to the "recipient" task, except:

  • if the sharing flow is prohibited,

  • if a restriction does not apply - For example, the source and beneficiary must be in the same organization or location, or the beneficiary must the source's supervisor etc.

A planner can look up the list of sharing commitments inferred by an information flow.
