Expressing exceptions by overriding tasks and flows

It is at times necessary to express "exceptions to the rule" when modeling IS practices. For example, IS practices might dictate that all employees alert their supervisors if they become aware of suspicious activities, whereas managers, who are also employees, should not alert their supervisors but rather directly contact the head of security.

Such exceptions in IS practices can be captured by defining overriding tasks and by defining prohibited flows from the overriding tasks that are otherwise equivalent to flows from the overriden tasks.

Figure 1.35. Overriding tasks and flows to express exceptions in IS practices


Task A overrides task B if:

  • they both have the same name (discounting uppercase vs. lowercase and extraneous blanks);

  • they are in the same segment or compatible segments (the segment of task A addresses the same situation as that of task B or a more specific version of it);

  • and the assignees to task A are a strict subset of the assignees of task B (all agents assignable to task A are also assignable to task B but there's fewer of them in task A).

When task A (for example, "managers detecting threats") overrides task B ("employees detecting threats") , agents assigned to task A are not assigned to the overridden task B. Furthermore, if task A is annotated as "prohibited," then no agent is assigned to task A and any agent assignable to task B is not assigned to task B either. This allows planners to express exceptions such as "all employees leave the building except for managers".

A flow A overrides another flow B if:

  • they both have the same name (dsicounting case and extra blanks)

  • the source task of flow A overrides the source task of flow B

  • and the target task of flow A is the same as or overrides the target of flow B

If an agent has sharing commitments computed from flow A then they replace any and all commitments for this agent computed from the overridden flow B. Furthermore, if overriding flow A is prohibited, then whoever would have commitments from flow A has commitments from neither flow A nor from overriden flow B.
