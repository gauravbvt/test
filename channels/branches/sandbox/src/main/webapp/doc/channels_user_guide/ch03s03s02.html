Specifying information sharing procedures

The IS procedures of an agent (a person or system) are the list of all tasks assigned to the agent, plus all consequent information sharing commitments as computed from an IS plan in which the agent participates. Planners directly set IS procedures. IS procedures are specified by the definitions of tasks and flows and by the profiles of entities (roles, organizations, agents, places, etc.)

The activities that change the set of IS procedures derived from an IS plan are:

  • Adding a task

  • Adding a task's information need

  • Adding a task's information capability

  • Connecting tasks with an information flow

Adding a task

A planner normally adds a task to a segment either because it

  • Directly achieves a goal listed in the definition of the segment

  • Produces information required by goal-achieving tasks

  • Is triggered by (or is ongoing at) the onset of the phase of the event the segment focuses on.

Table 3.8. Scoping: Add a task
Prerequisites:

  • The segment for the task was added.
Steps:

Note: All steps are optional.

  • Name what is to be done.

  • Categorize the task as:

    • operations (a task that directly accomplishes the mission of the organization)

    • operations management (a task to regulate and measure operations)

    • inter-operations coordination (a task to prevent or handle operational conflicts, without involving higher-ups)

    • direction (a management task tackling operational cohesion and optimization, resource allocation, setting performance targets)

    • audit (a management task that measures the organization's operational performance)

    • analysis (a task that examines and makes sense of the organization's external environment)

    • planning/preparing (a forward-looking task: market research, forward planning, strategy, R&D, etc.)

    • policy-setting (a task by the organization's ultimate authority: mission definition and evolution, ethos, balancing the demands of now vs. future).

  • Annotate the task (Is it operational? Is it prohibited?)

  • Set where the task is to be executed (a place or type of place).

  • Identify or describe who is to be assigned (by agent, role, jurisdiction, organization) and set whether assignees execute the task individually or as a team.

  • Name a significant event that can be initiated by doing the task (will be added to scope if first referenced).

  • Set the timing attributes (duration, periodicity, whether it starts with the segment or is ongoing).

  • List goals directly achieved by the task.

  • Provide free-form instructions.
Outcomes:

  • Information needs can be added to the task.

  • Information capabilities can be added to the task.

  • The task can be connected by information flows.

  • The list of agents to be assigned to the task can be computed.
