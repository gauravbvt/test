Add a task's information capability

The outgoing flows of a task implicitly define information capabilities (ability and willingness to share) for the task. By adding an explicit information capability to a task, a planner defines information produced or acquired by the execution of the task that may or may not be shared to satisfy the information needs of other tasks. The planner also indicates that the targets of outgoing flows of the same name are not necessarily the only acceptable tasks the information can be shared with. Channels looks at information needs and information capabilities of tasks to uncover missed opportunities for information sharing (see command "Satisfy all needs").

Note that only the explicit information capabilities of a task are visible across segments and offered as candidate sources for cross-segment flows.

Table 3.10. Scoping: Add a task's information capability
Prerequisites:

  • The task was added.
Steps:

(All steps are optional and they can be taken in any order)

  • Name the information

  • Note: tagging with an information standard tag enables auto-population of standard elements of information

  • Categorize the expected intent of sharing as one of:

    • report (summary of facts and conjectures: "Here's what we know...")

    • command (an injunction: "Do this!" "Stop that!")

    • announcement (declaration, communiqué, advertisement: “I-95 is congested”, “Spinach is safe”)

    • feedback (thumbs up or down, reward, punishment: "Objectives were met", "Report is incomplete")

    • alarm (signal of an unexpected change requiring outside assistance: "Distribution point is overwhelmed", “Levee is breached”)

  • Name the information's elements and for each one,

    • indicate if it is time-sensitive (i.e. are subject to timing requirements)

    • optionally give it a security classification

    • optionally describe it

    • optionally give special handling instructions

    • indicate, if applicable, how the element is a transformation of one or more elements of informations received by the task

  • Indicate whether sending the information would terminate this task.

  • State a relationship restriction on the target agent, if any. Namely the target agent must be

    • from the same overall organization

    • from a different overall organization

    • from the (exact) same organization

    • from a different (specific) organization

    • in the same location

    • in the same location and organization

    • in a different location

    • from a supervisor

    • from self

    • from someone else

  • List the alternate transmission channels to be used, in order of preference.

  • Give how quickly the information can be shared

    • If shared via notification, the time constraint gives the maximum delay from the task acquiring time-sensitive elements of information.

    • If shared via reply, the time constraint gives the maximum delay between request and reply.

  • Note: attaching a reference document of a graphic or video media type is highlighted.
Outcomes:

  • Whether the information capability is being exercised can be diagnosed.

  • Channels can automate the creation of flows that exercise the capability.

  • Channels can check whether outgoing flows of the same name conform to the expectations (channels, timing, elements etc.) expressed in the capability.

  • Channels can offer the task as a candidate source for the need of a task from another segment.
