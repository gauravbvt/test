Descriptions, instructions and attachments

Not everything in the definition of an IS plan can be captured neatly as "structured data". Some important data is best conveyed with free form text or via attached documents.

Most forms used to define elements of an IS plan contain a text field called "description" where planners can optionally enter explanatory or documentary text. Instead of a description field, tasks and flows have instruction fields. Descriptions and instructions are not examined by Channels (Channels is indifferent to their content) but they do appear in IS guidelines generated by Channels.

Figure 1.37. A segment, like most elements of an IS plan, has a description


Sometimes an element of a plan or the plan itself can be documented by a PDF or a Web page or can be illustrated by a photo or a video. Channels allows planners to attach documents to a plan element by uploading them or linking to them in the context of that element.

An attachment is optionally given a name (its URL is the default name). It is also given a type. Different kinds of plan elements allow different kinds of attachment types. Certain types of attachment have different properties.

The types of attachments recognized by Channels are:

  • Reference - Reference documentation. Reference documents attached to tasks and flows that are movies or graphics are specially highlighted.

  • Picture - A picture document is used to produce icons displayed in flow maps.

  • Policy - Document containing policy statements participants need to be aware of.

  • Mandating policy - Document containing policy statements that mandate tasks or flows.

  • Forbidding policy - Document containing policy statements that restrict tasks or flows.

  • MOU - Memorandum of understanding documenting sharing agreements.

  • Tags - Pre-defined tags (see below). Attachable to an IS plan as a whole.

  • Info standards - Pre-defined elements of informations implied by tags. Attachable to an IS plan as a whole.

  • Help - Used to provide support documentation

Figure 1.38. Media reference attachment


Media-type reference attachments are highlighted as icons that, when clicked, show the graphic or the video, assuming that the Web browser supports the media type.

Figure 1.39. Picture attachment and use as icon


Channels automatically converts pictures into icons it uses in various visualizations. Only one picture per plan element is allowed.

Figure 1.40. Prohibiting policy attachment and prohibited flow


Note that attaching a prohibiting policy document to a flow or a task does not in and of itself prohibits it. The policy might prohibit execution of the task or of the flow only in some but not all circumstances (and in ways too complex to be captured formally in the IS plan). To prohibit a flow or a task in an IS plan, planners must explicitly mark it as prohibited (via a check box in the task or flow form).
