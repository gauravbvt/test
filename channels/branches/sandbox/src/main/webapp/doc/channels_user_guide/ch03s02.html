Building an information sharing plan

An IS plan is built by information sharing planners (planners) who carry out activities defined in the ISP Method in order to move the plan toward completion and also increase its value.

Overview of the ISP Method activities

Activities alter the scope of an IS plan, bring the plan closer to completion, or improve the quality and value of the IS procedures generated from the plan.

Activities are organized into four groups:

  • Scoping an IS plan.

  • Specifying IS procedures by defining tasks and information flows from which the procedures are computed.

  • Profiling events, organizations, roles and other entities referenced by name in the IS plan.

  • Improving the IS plan by making it more manageable, robust, complete and reusable.

Scoping an IS plan:

Note: Scoping activities initiate the building of an IS plan but they can also be re-applied at any time, whenever the scope needs to change.

  • Localize the IS plan

  • Identify an event as incident

  • Identify a planned phase

  • Identify a participating organization

  • Identify a participating agent (as role-playing member of a participating organization)

  • Add a plan segment

  • Add an organization goal

  • Add a secrecy classification system

Note: Naming an ISP entity for the first time adds it to the IS plan. Removing all references to an ISP entity and emptying its definition removes it from an IS plan.

Specifying information sharing procedures:

Note: All "Add ..." ISP Method activities imply "modify" and "delete" counterpart activities.

  • Add a task (and specify its assignees)

  • Add a task's information need

  • Add a task's information capability

  • Connect a task to another with an information flow

Note: A "Connect ..." activity implies a counterpart "Disconnect..." activity.

Profiling referenced entities (actual and types) by assigning values to common and idiosyncratic attributes:

  • Profile an event (always a type)

  • Profile a phase (always a type)

  • Profile a role (always a type)

  • Profile an organization (actual or type)

  • Profile an agent (actual or type)

  • Profile a place (actual or type)

  • Profile a transmission medium (always a type)

Improving an IS plan by reorganizing/restructuring it to increase its manageability or applicability, or by making changes that may have pervasive impacts on the IS procedures generated from it:

  • Move tasks between segments

  • Generalize a task's definition

  • Change an entity's categorization

There is no fixed, prescribed order to the execution of ISP Method activities. Scope can be modified at any time while an IS plan is being built. Tasks and information flows can also be added, removed or modified at any point in time. Entities can be profiled as soon they are mentioned by name and they can be named at any time. An IS Plan can and should be improved early and often in order to keep it as manageable and as valuable as possible.
