Commands

Planners carry out ISP activities in Channels by executing Channels commands. Any change to an IS plan is the result of executing a command. Planners execute commands in Channels whenever they create and remove segments; when they add, cut, copy and paste tasks, needs, capabilities and flows; connect tasks with information flows, disconnect them, add an issue, remove a segment, move tasks across segments, add an intermediate to a flow, etc.

Some commands are directly triggered from “actions” menus within the context of a plan, segment, task, flow, or entity. Other commands are triggered by updating fields in edit forms, or by pressing buttons.

Some commands have simple effects (adding a task, for example) but others can be more complex:

  • “Disintermediate” removes a task that acts as a middleman in a flow

  • “Intermediate” adds a middleman task to a flow

  • “Satisfy all needs” automatically connects information needs of a task to matching but untapped information capabilities from other tasks

  • "Move tasks" moves one or more tasks from one segment to another, possibly also adding goals achieved by the moved tasks to the target segment

Complex commands are defined in terms of other commands that are executed completely or not at all. Most commands are reversible and can be undone and redone up and until some planner takes an action that would conflict with the undoing or redoing (for example, anyone removing a task prevents a planner from undoing a change he/she recently did to one of its attributes).

The execution of each command is journaled for replay should the server be interrupted and restarted. It is also recorded and listed in the activities tab in the collaboration panel. Channels makes sure that only one command is being executed on an IS plan at a time, no matter how many planners are working on a plan.
