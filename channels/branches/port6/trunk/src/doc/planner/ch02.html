Channels is a collaborative Web application information sharing planners use to build, analyze and improve information sharing plans (IS plans), and to publish information sharing protocols (aka procedures or guidelines).

A Channels server can host multiple IS plans. Users can be given access to more than one plan hosted on a Channels server.

IS plans are versioned. Each plan has one development version being developed, possibly one production version that can no longer be changed, and zero or more retired versions (prior production versions).

User Roles

Channels recognizes three user roles: administrator, planner, and participant.

A Channels server has at least one administrator. An administrator has unlimited rights and privileges over the Channels server and the plans hosted on it. A user is given planner privileges by an administrator for some or all plans on a Channels server. Planners jointly develop, review, and deploy IS plans for which they have been given planner status by an administrator.

Users can be given participant access to some or all plans by an administrator. Participants view their IS guidelines and respond to surveys from planners (currently only about issue resolution). A user can be a planner for some plans, a participant for others, and has no access to plans for which he/she is neither planner nor participant.

An administrator implicitly also has planner privileges over all plans hosted on the Channels server. A planner for a given plan implicitly also has participant privileges.

An administrator of a Channels server can:

  • configure the Channels server

  • add new plans

  • delete existing plans

  • move a development version of a plan into production, retiring the previous production version, if any

  • assign users to plans and change their user roles

A planner of an IS plan (and all administrators) can:

  • view all non-retired versions of the plan

  • modify the development version

  • associate participants with agents (production and development versions only)

  • view plan analytics

  • vote with other planners to put the development version into production

  • launch, access, and close online surveys of selected planners and participants about issues in the plan

A participant in an IS plan (and all its planners and all administrators) can:

  • view the IS guidelines report for the agent he/she is associated with

  • provide feedback about his/her IS guidelines

  • access and respond to online surveys

  • if a version of a plan is marked as a "viewable by all"

    • view (but not modify) the details of that version

    • view all analytics for that version

    • forward a survey to a non-user and, in the process, grant participant privilege for the plan

Note that a participant in an IS plan does not have IS guidelines until associated with an agent in the plan. A participant can self-associate to agents that have been marked by planners as "open" agents.
